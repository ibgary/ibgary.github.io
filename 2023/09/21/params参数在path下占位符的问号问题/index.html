关于params参数在path下占位符的问号问题

在 Vue Router 中，路由路径中带有问号 ? 的参数是可选的。这意味着在访问路由时，该参数可以存在也可以不存在，并且不会影响路由的匹配。

例如，在以下路由路径中：

keyword 参数后面的问号表示该参数是可选的。这意味着可以通过以下 URL 来访问这个路由：

  • /search
  • /search/vue
  • /search/react

在第一个 URL 中，keyword 参数不存在。在第二个和第三个 URL 中，keyword 参数分别为 vue 和 react。

当您使用带有可选参数的路由路径时，可以在组件中使用 $route.params.keyword 来访问参数的值。如果参数不存在，那么 $route.params.keyword 将会是 undefined。

如果路由路径中没有问号 ?，则表示该路由参数是必须的，访问该路由时必须提供该参数的值。

在访问 /users/123 时，:id 参数是必须提供的，否则该路由将无法匹配。

在组件中，您可以通过 $route.params.id 来访问该参数的值。如果路由路径中的参数是必需的，则可以在组件中将其视为必需属性，从而确保该属性始终存在。
