二级路由child的命名规则

发表于2023-03-18|更新于2023-03-18|tech
|阅读量:

Q : 为什么在一个存在二级路由的router中vue提示我删除name值？

A : 在 Vue Router 中，如果你使用了命名路由，并且这些命名路由是作为父路由配置的子路由，那么你在父路由配置中定义的 name 值会被 Vue Router 自动为子路由生成一个前缀。这个前缀的格式是 parentName-childName，其中 parentName 是父路由的 name 值，childName 是子路由的 name 值。

例如，假设你有如下的父路由配置和子路由配置：

在这个配置中，父路由的 name 值为 'parent'，子路由的 name 值为 'child'，那么子路由的实际 name 值将是 'parent-child'。这个前缀的作用是避免命名冲突，因为在同一个应用程序中可能会存在多个父子路由配置。

所以，如果你在一个存在二级路由的 Vue Router 中定义了父路由的 name 值，并且同时给子路由配置了 name 值，Vue 会提示你删除子路由的 name 值，因为它们实际上是有前缀的，这样可以避免命名冲突。如果你非常确定子路由的命名不会与其他命名路由冲突，你可以选择忽略这个警告，但这并不是推荐的做法。
